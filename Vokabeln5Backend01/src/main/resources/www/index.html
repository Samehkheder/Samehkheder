  • Read the docs

Voc6

Das Backend wurde mit dem Spark Java Framework geschrieben. Es liest Daten aus einer PostgreSQL Datenbank und liefert sie an Clients aus. Des Weiteren erlaubt es Daten zu löschen und zu modifizieren. Die genauen Operationen, die es unterstützen soll finden Sie unten in der Dokumentation.

  • Get Started

GET /login

GET calls auf /login werden mit status 200 oder 401 beantwortet je nachdem, ob die Header email und password valide Werte haben, oder das JWT valide ist.

POST /register

POST calls auf /register, die in dem Body ein valides JSON haben, soll zur Folge haben, dass ein neuer Nutzer erstellt wird. Das Objekt im Body soll zwei Strings enthalten: email und password. Ist der Nutzer bereits vorhanden, wird ein Fehler zurückgegeben.

GET /voc

Alle Vokabeln aus der Datenbank werden als JSON ausgegeben.

GET /voc/random

Eine zufällige Vokabel (eine, die noch nicht Phase 5 hat) wird als JSON zurückgegeben.

POST /voc

Übergibt der Nutzer hier ein JSON Objekt einer Vokabel als Body, wird diese in die Datenbank eingetragen.

PATCH /voc/:id

Übergibt der Nutzer hier ein JSON Objekt einer Vokabel als Body, sollen alle Felder, die nicht null sind, in der Datenbank überschrieben werden. Die ID der Vokabel ergibt sich aus dem Pfad.

DELETE /voc/:id

Ruft der Nutzer diese URL auf, wird die referenzierte Vokabel gelöscht.

GET /login/jwt

Sind Email und Password richtig gesetzt, wird ein signierter JWT Token ausgehändigt. Dieser hat eine Laufzeit von 3 Monaten. In einem JWT Token wird von dem Server die Identität des Nutzers kryptographisch unterschrieben.

POST /changePassword

Nach einer validen Anmeldung, wird der Wert aus dem Body als neues Passwort gesetzt.

POST /changeEmail

Nach einer validen Anmeldung, wird der Wert aus dem Body als neue Email gesetzt.
